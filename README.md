# New-API 每小时报表 · AstrBot 插件

每小时自动生成 New-API 监控报表，含请求趋势图、模型/用户 Token 排行、服务器状态，发送到 QQ 群。

> 致谢：本插件基于 [huanxherta](https://github.com/huanxherta) 的原始脚本改造而来。

## 功能特性

- 📊 **请求趋势图**：30 分钟一档的柱状图，时间范围可配置（1-48 小时）
- 🏆 **模型 Token 排行**：Top 10 模型消耗排行
- 👤 **用户/IP Token 排行**：按用户汇总，底部明细按账号完整展示近 24 小时全部 IPv4 / IPv6
- 🖥️ **服务器状态**：CPU、内存、Swap、磁盘、网络、负载、温度、IO、进程等
- ⏰ **定时发送**：每小时自动生成并发送到指定 QQ 群
- 📨 **多群发送**：支持同时发送到多个群
- 🎨 **随机背景**：每次生成报表重新请求背景图，失败时自动复用旧缓存
- 🖼️ **多种底图源**：支持 loliapi、Lolicon/Pixiv、两者随机和本地图片，并支持回退链
- ⚡ **底图预加载**：参考状态插件，可后台预取底图并为在线请求单独配置代理
- 🔞 **R18 选项**：Lolicon 来源支持正常、R18、混合三种模式
- 🔍 **清晰渲染**：使用 2 倍像素密度、紧凑裁切和大号文字，提升 QQ 查看时的可读性
- 📐 **近 24 小时概览**：展示请求效率、上榜账号、完整 IP、峰值时段、模型占比和用量集中度，填充双栏空区
- 🔄 **API 重试**：网络波动自动重试，3 次递增退避
- ✅ **完整增量同步**：从上次成功游标回退 5 分钟重拉；任一分页失败时不推进游标
- 👁️ **IP 显示开关**：总开关、IPv4、IPv6 均可独立控制，关闭时不向报表 HTML 写入对应地址
- 🧩 **清晰配置分组**：连接、发送、显示、背景和高级选项分组展示
- 🖼️ **长图背景适配**：默认按宽度适配并镜像交替平铺，超长报表也能自然衔接

1. 在 AstrBot 插件市场搜索 `newapi_report` 或手动安装：

## 配置

配置页已按用途分为：

- **New-API 连接**：地址、系统访问令牌、管理员用户 ID
- **定时发送**：群号、定时开关、执行分钟
- **报表显示**：统计范围、IP 总开关、IPv4 和 IPv6 开关（全部默认开启）
- **背景图**：主来源、回退链、本地路径、预加载、超时、代理、R18 和长图适配
- **高级设置**：Chromium 路径

IP 显示开关全部默认开启：

- `显示 IP 地址`：总开关，关闭后隐藏全部 IP 区域
- `显示 IPv4`：关闭后仅过滤 IPv4，IPv6 仍可显示
- `显示 IPv6`：关闭后仅过滤 IPv6，IPv4 仍可显示

背景图长图适配方式：

- `repeat`：原图与上下翻转图镜像交替平铺，接缝自然，默认且推荐用于超长报表
- `cover`：使用单张图片铺满整张报表
- `contain`：完整显示单张图片，剩余区域使用渐变底色

底图来源配置参考状态插件：

- `背景获取来源（主）`：选择 loliapi、Lolicon、两者随机、本地图片或渐变背景
- `背景回退链`：主来源失败后按顺序尝试其他来源
- `预加载背景数量`：后台提前获取底图，减少生成报表时的等待
- `背景请求代理`：仅用于在线底图请求，不影响 New-API 请求

### 获取系统访问令牌

1. 登录 New-API 管理后台
2. 个人设置 → 安全设置 → 系统访问令牌
3. 生成令牌并填入配置

> **注意**：需要管理员账号生成令牌，同时需填写管理员用户 ID（通常为 1）。

### IP Token 排行

IP 排行功能需在 New-API 后台「个人设置 → Other Settings → IP Logging」中手动开启，默认关闭。

## 使用

- **手动触发**：在群内发送 `/报表`
- **发送当前底图**：管理员发送 `/报表底图` 或 `/报表原图`
- **定时发送**：配置 `enable_cron=true` 后，每小时第 N 分钟自动发送

## Docker 环境注意事项

插件依赖 Playwright 无头浏览器截图，Docker 容器内需安装系统依赖：

```bash
# 进入 AstrBot 容器
docker exec -it astrbot bash

# 安装 Playwright + Chromium
pip install playwright
npx playwright install chromium
npx playwright install-deps chromium
```

也可配置 `browser_executable` 指向容器内已有的 Chromium：
```
/usr/bin/chromium
```
